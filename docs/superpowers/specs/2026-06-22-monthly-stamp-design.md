# 월간 도장 시스템 (Monthly Stamp)

작성일: 2026-06-22

## 목적

훈련일지를 안 쓰거나 건성으로 쓰는 수강생을 줄이기 위해, **코치가 한 달에 한 번 일지에 등급 도장을 찍는다.** 학생은 "참 잘했어요" 도장을 받고 싶어서, 또 "코치가 내 일지를 보고 있다"는 감시·관심 효과 때문에 더 성실히 기록하게 된다.

핵심 제약: **코치가 일지를 일일이 안 읽고도** 도장을 찍을 수 있어야 한다.

## 3등급

| 등급 키 | 도장 문구 | 색 (hex) |
|---|---|---|
| `great` | 참 잘했어요 | 빨강 `#E94E58` |
| `good` | 잘하고 있어요 | 코발트 `#329BE7` |
| `tryharder` | 더 힘내요! | 앰버 `#EDBC40` |

> 훈련일지는 별도 HTML(`:root` 변수 못 씀)이라 hex 직접 사용.

## 데이터 모델

Firebase 컬렉션 `monthlyStamps` 신규.

- 문서 ID: `{userName}__{YYYY-MM}` (예: `홍길동__2026-06`)
- 한 학생 × 한 달 = 문서 1개

```
{
  userName: string,
  month: 'YYYY-MM',            // 도장을 찍은(부여한) 달
  grade: 'great' | 'good' | 'tryharder',
  comment: string,            // 코치 한 줄 코멘트 (선택)
  stampedBy: string,          // 코치 이름
  stampedAt: ISO string,
  seenByStudent: boolean      // 학생이 도장 팝업을 봤는지 (첫 접속 1회 팝업용)
}
```

- `month`은 **도장을 부여한 달**. 활동 통계는 기존 티어 관례대로 **지난달** 기록을 본다(코치가 6월 1일에 5월 훈련을 평가 → `month: '2026-06'`). 학생에게는 "6월의 도장"으로 노출.

## 코치 흐름 (부담 최소화 — 핵심)

훈련일지 코치 화면의 **"운동 종목 관리" 버튼 옆에 `📋 이달의 도장` 버튼** 추가. 누르면 기존 일지 수정 모달과 같은 오버레이/컨테이너를 재사용한 모달이 뜬다(새 모달 프레임워크 만들지 않음). 모달은 `coach.js`에서 HTML 문자열로 생성, 핸들러는 `window.*` 노출(main.js `Object.assign(window, Coach)` 패턴), 색/도장칩은 hex 인라인.

모달 레이아웃:

```
┌─────────────────────────────────────────────┐
│  📋 6월의 도장              지난달(5월) 기준  ✕ │
├─────────────────────────────────────────────┤
│  이름    활동일  일평균종목   등급             │
│  홍길동   18일   4.2 ✓     [🔴참잘했어요 ▼]   │
│   └ 💬 [ 이번 달 정말 꾸준했어요!______ ]      │
│  김철수   16일   1.1 ⚠️    [🔵잘하고있어요▼]   │
│   └ 💬 [ 종목 수를 좀 더 채워봐요___ ]         │
│  이영희    3일   3.0       [🟡더힘내요  ▼]     │
│  ...                                         │
│              [ 전체 확정 (28명) ]             │
└─────────────────────────────────────────────┘
```

- 모달 열릴 때 학생별 지난달 `records` 1회씩 조회 → 활동일·일평균 종목 계산 → 등급 드롭다운 자동 프리필, 일평균 1점대면 `⚠️` 표시.
- 각 행에 **고정 메모 개수(`📝 메모 N개`)** 표기(`pinnedMemos` 컬렉션 전체 1회 조회). 클릭하면 그 학생의 메모 목록(운동명+내용, 통증 ⚠️)이 펼쳐진다 — 코치가 메모 품질로 성실도를 판단할 보조 신호. 개수는 전체 고정 메모 수(월 한정 아님).
- 이미 이번 달 도장 있으면 그 값으로 채워 열림(재수정).
- **[전체 확정]** = `monthlyStamps` batch write 한 번.

열면 수강생 전원이 한 리스트에 뜬다(위 모달 내부):

```
홍길동   활동 18일 · 일평균 4.2종목   [참잘했어요▼]   💬____
김철수   활동 16일 · 일평균 1.1종목   [잘하고있어요▼]  💬____  ← 건성(일평균 낮음)
이영희   활동  3일 · 일평균 3.0종목   [더힘내요▼]      💬____
        ...
                            [전체 확정]
```

- **자동 추천 등급**: 지난달 활동일 수로 미리 채워둠. 경계는 기존 티어 경계 재사용
  (활동일 ≥ 13 → `great`, ≥ 6 → `good`, 그 미만 → `tryharder`). 구현 시 숫자는 한 곳 상수로.
- **보조 신호 — 일평균 기록 종목 수**: `records` 컬렉션에서 (지난달 총 기록 종목 수 ÷ 활동일). "활동일은 높은데 일평균 1종목대"인 건성 케이스가 한눈에 튄다. **자동 페널티는 걸지 않음** — 코치 눈에만 띄게 하고 판단은 사람이 한다(헬스 하루 2종목 빡세게 하는 사람 오판 방지).
- 코치는 추천을 훑고 이상한 줄만 등급 변경 / 코멘트 입력 → **[전체 확정]** 한 번으로 그 달 전원 `monthlyStamps`에 일괄 저장(batch write). 이미 도장 있는 달이면 그 값으로 프리필되어 재수정 가능.

## 코치 알림 (빨간점)

- 메인 React 앱 `BottomNav.jsx`의 **"훈련일지" 탭**에 빨간점 표시.
- 조건: **코치 모드 + 이번 달 `monthlyStamps` 미완료**. 매달 1일에 자연히 켜지고, 코치가 그 달 도장을 확정하면 사라진다.
- 판정: 이번 달(`YYYY-MM`) 문서 존재 여부 1회 조회. (수강생 전원분이 생겼는지까지 보지 않고, "이번 달 도장 작업을 했는가"만 본다 — 한 번 [전체 확정]하면 꺼짐.)

> ponytail: "전원 다 찍었나"가 아니라 "이번 달 작업했나"만 본다. 미세하게 빠진 학생은 코치가 알아서. 정밀 카운트는 필요해지면 그때.

## 학생 흐름

- 훈련일지 상단에 **이번 달 도장 배지**(문구 + 색) 노출. 도장 밑에 코치 코멘트 있으면 함께.
- **도장 받은 후 첫 접속 팝업**: 학생이 훈련일지에 들어왔을 때 이번 달 도장의 `seenByStudent === false`면 도장 팝업을 띄우고, 닫으면 `seenByStudent = true`로 업데이트. (기존 티어 인트로 팝업 `tierIntroPending` 패턴과 동일.)
- 도장 없는 달이면 배지·팝업 모두 없음.

## 재사용 / 참고

- 자동 추천·활동일·지난달 범위: 티어 로직(`src/utils/tiers.js` `computeActiveScore`, 경계 상수)과 같은 개념. 훈련일지는 별도 앱이므로 활동일/종목 집계는 training-log 쪽 `records` 조회로 자체 계산하되 경계값만 맞춘다.
- 코치 코멘트 UI/저장: 기존 `saveCoachCommentToStudentMemo()`(coach.js) 패턴 참고.
- 학생 팝업: 메인앱 `TierChangeModal.jsx` / `tierIntroPending` 흐름 참고(개념만 — 구현은 training-log 내부).

## 범위에서 제외 (YAGNI)

- 도장 히스토리/누적 컬렉션 화면 — 이번 달 1개만 보이면 충분. 필요해지면 추가.
- 일평균 종목 수 자동 페널티 / 임계 경고 — 코치 판단에 위임.
- 학생에게 도장 푸시/문자 알림 — 앱 내 배지·팝업으로 충분.
- 도장 등급별 점수·랭킹 연동 — 별개 기능.
