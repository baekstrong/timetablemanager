# 입학반 수동 등록 + 관련 버그 수정 설계

> 작성일: 2026-04-22
> 대상 컴포넌트: `StudentRegistrationModal`, `CoachNewStudents`

## 배경

코치가 신규 수강생을 직접 추가/관리할 때 다음 문제가 있다.

1. 상담 등으로 직접 등록하는 신규 수강생을 입학반과 연결할 방법이 없다. 현재 `StudentRegistrationModal`(수강생 관리 페이지)은 Google Sheets에 행을 추가하지만 입학반(`entranceClassId`)과의 연결은 전혀 하지 않는다.
2. `CoachNewStudents` → 입학반 관리 → "수강생 추가" 모달이 이미 입학반을 마친 신규 수강생도 계속 노출한다. 시트 F열의 '신규' 표시는 재등록 전까지(약 1개월) 유지되고, 모달 필터가 "이 입학반에 미배정"만 보기 때문이다.
3. 지난 입학반 카드가 입학반 관리 화면에 그대로 노출되어 시야를 어지럽힌다.
4. 코치가 직접 등록한 신규 수강생에게는 안내 SMS가 전혀 발송되지 않는다.

## 목표

- 코치가 `StudentRegistrationModal`에서 신규 수강생을 등록할 때 입학반을 함께 선택할 수 있게 한다.
- 직접 등록 흐름에서도 학생에게 적절한 안내 SMS(승인 + 입학반 리마인더)가 발송되게 한다.
- 입학반 관리 UX의 잡음을 줄인다(이미 입학반을 마친 학생 노출 제거, 지난 입학반 접힘 처리).

비목표:
- `?register=true` 학생 자가 등록 흐름은 변경하지 않는다.
- 재등록 흐름의 SMS는 변경하지 않는다.
- 새로운 SMS 템플릿을 만들지 않는다(기존 SMS 2/3 재사용).

## 변경 사항

### A. `StudentRegistrationModal` — 입학반 선택 + SMS 발송

#### A1. 입학반 선택 필드 추가

신규 모드(`registrationType === 'new'`)에서만 표시한다.

- 위치: 결제 정보 위쪽(이름/시간표 섹션 아래, 예정 출석일 달력 위 근처).
- 컨트롤: `<select>` 기반.
- 옵션:
  - `선택 안 함` (value: `''`)
  - 다가오는 입학반 목록: `오늘 ≤ ec.date` 이고 `ec.isActive === true`인 항목, `date` 오름차순.
  - 만석(`currentCount >= maxCapacity`)인 항목은 `disabled` 처리하고 라벨에 `(만석)` 접미사.
- 라벨 형식: `M월 D일(요일) HH:MM ~ HH:MM (N/M명)`. `formatEntranceDate(ec.date)` 활용.
- 기본값: 다가오는 활성 입학반 중 만석이 아닌 것 중 `date`가 가장 빠른 것의 `id`. 없으면 `''`.
- 데이터 로드: 모달 마운트 시 `getEntranceClasses(false)` 호출. 이미 시트/공휴일 로드와 같은 `useEffect`에 추가하거나 별도 effect.

재등록 모드에서는 필드 자체를 렌더링하지 않는다.

#### A2. 등록 시 입학반 연결

`handleSubmit` 안에서 시트 행 작성과 `createNewStudentRegistration` 호출 사이/직후 처리:

1. `selectedEntranceId`가 있으면 해당 `ec` 객체를 찾는다.
2. `createNewStudentRegistration` 호출 시 `entranceClassId`, `entranceDate`, `entranceClassDate` 필드를 함께 저장한다.
   - `entranceClassDate`: `${formatEntranceDate(ec.date)} ${ec.time}${ec.endTime ? ' ~ ' + ec.endTime : ''}`
3. `updateEntranceClass(ec.id, { currentCount: (ec.currentCount || 0) + 1 })` 호출.
4. 입학반이 그 사이 만석이 됐으면 알리고 연결 없이 진행할지 확인(현재 컨텍스트 단순화를 위해 그냥 인원 ≥ 최대 시 alert + 등록 중단).

#### A3. SMS 발송 (옵션 A)

신규 모드 + `form.핸드폰`이 있을 때만 발송한다. 시트 쓰기/Firestore 저장이 모두 성공한 후에 호출하며, 실패해도 등록 자체는 성공으로 본다(기존 패턴과 동일).

```js
import { sendApprovalNotifications } from '../services/smsService';

// 결제방식 한글 → SMS 코드 매핑
const PAYMENT_CODE_MAP = {
  '네이버': 'naver',
  '카드': 'card',
  '제로페이': 'zeropay',
  '계좌': 'cash'
};

const smsResults = await sendApprovalNotifications(form.핸드폰, form.이름, {
  paymentMethod: PAYMENT_CODE_MAP[form.결제방식] || 'cash',
  weeklyFrequency: parseInt(form.주횟수) || 0,
  scheduleString: form['요일 및 시간'],
  entranceDate: ec ? ec.date : '',
  entranceClassDate: ec ? `${formatEntranceDate(ec.date)} ${ec.time}${ec.endTime ? ' ~ ' + ec.endTime : ''}` : ''
});

// 예약 SMS groupId 저장 (취소 가능하게)
const groupId = smsResults.reminderSMS && typeof smsResults.reminderSMS === 'object' ? smsResults.reminderSMS.groupId : null;
if (groupId) {
  await updateNewStudentRegistration(createdRegId, { reminderGroupId: groupId });
}
```

`createNewStudentRegistration`은 이미 `{ success, id }`를 반환하므로 그 `id`를 그대로 사용한다.

코치 SMS, SMS 1(접수)는 발송하지 않는다.

### B. `CoachNewStudents` — 수강생 추가 모달 개선

#### B1. "+" 직접 등록 버튼

`수강생 추가` 모달 푸터(`cns-modal-actions`)의 "취소"/"추가" 옆에 한 개 버튼 추가:

- 라벨: `+ 직접 등록`
- 동작: 클릭 시 `setShowAddStudentModal(null)` 으로 현재 모달 닫고, `StudentRegistrationModal`을 신규 모드로 연다.
  - 새 state: `const [showRegModalForEC, setShowRegModalForEC] = useState(null)` (현재 입학반 컨텍스트 보존).
  - `<StudentRegistrationModal initialEntranceId={showRegModalForEC.id} ... />` 형식으로 prop 전달.
- `StudentRegistrationModal`에서 `initialEntranceId` prop을 받으면 입학반 셀렉터 기본값을 그 id로 세팅한다.
- 등록 성공(`onSuccess`) 시:
  1. 등록 모달을 닫는다.
  2. `loadEntranceClasses()` 호출로 인원/배정 정보 갱신.
  3. 수강생 추가 모달은 다시 열지 않는다(사용자가 필요시 다시 클릭).

#### B2. 필터 강화 — "이미 지난 입학반에 다녀온" 신규 학생 제외

`handleOpenAddStudent` 내부:

- 진입 직후 `getNewStudentRegistrations(null)`로 모든 등록 이력을 한 번 가져온다(`entranceRegs`는 status !== 'rejected'만 있어 부족함).
- `today` 자정 기준으로, `r.entranceClassId`가 있고 `r.entranceDate`가 과거인 등록의 `name`을 모은 `Set`을 만든다.
- 시트 신규 학생 루프에서 `existingNames.has(name)` 또는 `completedEntranceNames.has(name)` 시 스킵.

```js
const today = new Date(); today.setHours(0, 0, 0, 0);
const allRegs = await getNewStudentRegistrations(null).catch(() => []);
const completedEntranceNames = new Set();
for (const r of allRegs) {
  if (!r.name || !r.entranceClassId || !r.entranceDate) continue;
  const d = new Date(r.entranceDate + 'T23:59:59');
  if (d < today) completedEntranceNames.add(r.name);
}
// 루프 내부:
if (completedEntranceNames.has(name)) continue;
```

### C. 입학반 관리 — 지난 입학반 접힘 섹션

- `entranceClasses`를 렌더 직전에 `upcoming`(오늘 ≤ ec.date) / `past`(ec.date < 오늘)로 분리한다.
- 정렬:
  - `upcoming`: `date` 오름차순.
  - `past`: `date` 내림차순(최근부터).
- 렌더:
  - 상단: `upcoming` 카드 목록.
  - 하단: 토글 버튼 `▼ 지난 입학반 ({past.length})` (없으면 토글 자체를 렌더하지 않음). 클릭 시 펼쳐 `past` 카드 목록을 표시.
- 새 state: `const [showPastEntrance, setShowPastEntrance] = useState(false);`.
- 비활성 뱃지(`cns-inactive-badge`)는 그대로 유지.
- CSS 추가: `.cns-past-entrance-toggle` (가로폭 100%, 좌측 정렬, 회색 배경, 호버 시 강조).

## 데이터 흐름

```
[코치] StudentManager → "+ 등록" → StudentRegistrationModal (신규 모드)
   │
   ├─ 입학반 셀렉터: 다가오는 EC 목록 (기본=가장 가까운 것)
   │
   └─ 제출
        ├─ Sheets 행 추가 + 음영 적용
        ├─ Firestore newStudentRegistrations 생성 (status='approved')
        │     └─ 입학반 선택 시: entranceClassId/entranceDate/entranceClassDate 함께 저장
        ├─ 입학반 currentCount += 1 (선택 시)
        └─ SMS 2 발송 + (선택 시) SMS 3 예약 → groupId 저장
```

```
[코치] CoachNewStudents → 입학반 관리 → 입학반 카드 "+" → 수강생 추가 모달
   │
   ├─ 시트 신규 학생 리스트 (기제외: 본 입학반에 이미 배정된 자, 지난 입학반 다녀온 자)
   ├─ 학생 선택 후 "추가" → 기존 동작 유지
   └─ "+ 직접 등록" 클릭 → StudentRegistrationModal (입학반 prefill) → 위와 동일 흐름
```

## 에러/엣지 케이스

- **다가오는 입학반 0개**: 입학반 셀렉터 기본값은 빈 값. 학생은 입학반 미연결로 등록 가능. SMS 2만 발송.
- **선택한 입학반이 동시 등록으로 만석**: 등록 직전 다시 조회해서 만석이면 alert + 등록 중단.
- **SMS 설정 누락**: 콘솔 경고만, 등록은 성공으로 처리(현재 패턴 동일).
- **결제방식 미선택**: SMS 2 결제 안내가 "입학반날 방문하셔서 결제해주세요!"로 fallback. 기존 SMS 코드가 이미 처리.
- **수강생 핸드폰 없음**: SMS 발송 스킵.
- **시트 쓰기는 성공했지만 입학반 currentCount 업데이트 실패**: 콘솔 경고만, 등록은 성공. 코치가 인원 수동 수정 가능.

## 테스트 계획 (수동)

코드를 변경한 뒤 다음 시나리오를 사용자가 직접 확인:

1. 수강생 관리에서 신규 수강생 등록 → 입학반 셀렉터 기본값이 가장 가까운 다가오는 입학반인지 확인.
2. 입학반 선택 후 등록 → Firestore `newStudentRegistrations`에 `entranceClassId` 저장됨, 입학반 `currentCount` 증가, 학생 핸드폰으로 SMS 2 도착, 입학반 3일 전 리마인더 예약됨(또는 즉시 발송).
3. "선택 안 함"으로 등록 → SMS 2만 도착(입학반 정보 줄 없음), 리마인더 예약 없음.
4. 입학반 관리 → "수강생 추가" 모달 → 이미 지난 입학반에 다녀온 신규 학생(예: 정윤희, 주상조)이 더 이상 보이지 않음.
5. 동일 모달의 "+ 직접 등록" 버튼 → StudentRegistrationModal이 열리고 입학반 셀렉터 기본값이 해당 입학반으로 세팅됨.
6. 입학반 관리 화면 → 지난 입학반 카드가 하단 접힘 섹션에 모여 있고, 펼치면 표시됨.
