# 매출·통계 대시보드 개선 (v2) 설계

작성일: 2026-06-11
대상: 근력학교 수강 관리 시스템 — `AnalyticsDashboard`

## 1. 목적

기존 매출·통계 대시보드의 3가지를 개선한다.
1. **레이아웃 정리** — 현재 '이번 달 현황'과 '6개월 추세'가 섞여 있어, 위=현황 / 아래=추세로 그룹화.
2. **월 선택** — 상단 '월별 현황' 섹션을 선택한 달 기준으로 조회.
3. **환불 표시** — 선택한 달 환불액(현황 KPI) + 6개월 환불 추세(추세 차트).

## 2. 배경 데이터 (라이브 시트 확인 완료)

시트 우측 '엑셀 시트 집계' 블록(행2=라벨, 행3=값, 모두 **만원 단위**, 환불은 음수):

| 열 | 라벨 | 비고 |
|---|---|---|
| T | 계좌 | 결제방식 금액 |
| U | 카드 | |
| V | 네이버 | |
| W | 탈잉 | |
| X | 제로페이 | |
| Y | 어플 | |
| AD | 총 매출 | |
| AE | **환불** | 음수·소수 (예: `-53.025`) |
| AF | **최종 매출(환불 포함)** | 현재 매출 지표로 사용 중 |

→ `T2:AF3` 한 번 읽으면 결제방식 금액 + 환불 + 최종 매출을 동시에 취득.

## 3. 레이아웃 (재배치)

```
[월 선택기]                              ← 상단, '월별 현황'만 제어
── 월별 현황 (선택한 달) ──
KPI 행: 매출 · 전월 대비 · 환불 · 총 수강생 · 신규/재등록/이탈
도넛/막대: 결제방식(금액) · 남녀 비율 · 유입 경로 · 직업 비율
── 6개월 추세 (항상 최근 6개월, 고정) ──
막대: 매출 추세 · 환불 추세 · 이탈 추세
```

- 두 섹션을 `<h2>` 헤더(`월별 현황 (N월)`, `6개월 추세`)로 명확히 구분.
- 새로고침 버튼은 그대로 상단 우측(전체 재조회).

## 4. 월 선택기

- 범위: **최근 6개월**(추세에 쓰는 6개월과 동일), 기본값 = 이번 달.
- 변경 시 **'월별 현황' 섹션 데이터만** 재조회. 6개월 추세·환불·이탈 차트는 불변.
- 구현: 대시보드 로컬 `selected` state(`{year, month}`). 전역 `GoogleSheetsContext.selectedMonth`는 건드리지 않음(다른 화면 영향 방지). 간단한 `<select>` 드롭다운(최근 6개월 목록). 기존 `MonthSelector.jsx`는 전역 상태에 묶여 있어 재사용하지 않음.

## 5. 환불

- **현황 KPI**: 선택한 달 환불액 = `|AE3|` × 10000 (원). 0이면 `0원`.
- **추세 섹션**: 최근 6개월 각 달 `AE3`(만원·음수) → `|값|`×10000 원으로 막대 차트.

## 6. 결제방식 = 금액

- 시트 집계 블록 `T3:Y3`(계좌/카드/네이버/탈잉/제로페이/어플, 만원) → ×10000 원.
- 금액 0인 방식은 차트에서 제외.
- 라벨은 행2(`T2:Y2`)에서 동적으로 읽어 매핑(블록 라벨 변동 대비).
- 도넛 차트 value = 금액(원). 툴팁에 원화 표시.

## 7. 데이터 계층 (`analyticsService.js` 리팩토링)

기존 `buildDashboard(6)`를 두 함수로 분리하고, 컴포넌트가 조합한다.

### 7.1 `getTrends(monthsCount = 6, baseDate = new Date())`
최근 N개월(고정) 추세. 반환:
```
{ months: [{year,month}...],
  revenueTrend: [{year,month,revenue,delta,deltaPct}...],   // AF3
  refundTrend:  [{year,month,refund}...],                   // |AE3|×10000
  churnByMonth: { 'YYYY-MM': count } }                       // 기존 로직(시트+Firebase)
```
- 월별로 `AE3:AF3` 읽기 + `getAllStudents`(이탈 판정용) + Firebase(terminations).
- `computeRevenueTrend`, `computeSheetChurnByMonth`, `mergeChurn` 재사용.

### 7.2 `getMonthSnapshot(year, month, registrations)`
선택한 달 현황. 반환:
```
{ year, month,
  revenue,                 // AF3 ×10000
  prevDelta: {delta, deltaPct},  // 전월 AF3 대비
  refund,                  // |AE3| ×10000
  totalStudents,
  newVsRenewal: {신규,재등록},
  payments,                // { 방식: 금액(원) }  ← 집계블록 T3:Y3
  genders, occupations,    // 해당 달 students
  referrals }              // 해당 달 생성 registrations
```
- 집계 블록 1회 읽기(`T2:AF3`)로 payments+refund+revenue.
- 전월 대비: 전월 `AF3` 1회 읽기 → `computeRevenueTrend`와 동일 공식.
- `getAllStudents(year,month)` → 총원/신규·재등록/남녀/직업.
- `referrals`: 인자로 받은 전체 `registrations`를 `createdAt`이 해당 (year,month)인 것만 `tallyReferralSources`.
- **이탈은 여기서 계산하지 않음** — 컴포넌트가 `trends.churnByMonth['YYYY-MM']`로 채움(추세에서 이미 계산).

### 7.3 집계 블록 파서 (신규 순수 함수, 단위 테스트)
`parseAggregateBlock(labelRow, valueRow)` →
```
{ payments: {계좌:원, 카드:원, ...(0 제외)}, refund: 원(양수), finalRevenue: 원 }
```
- 만원→원(×10000), `parseFloat`로 음수·소수 보존, refund는 절대값.
- 라벨·값 zip, 알 수 없는 라벨/빈 값 무시.

## 8. 컴포넌트 (`AnalyticsDashboard.jsx`)

- state: `trends`, `snapshot`, `selected({year,month})`, `loading`, `snapLoading`, `error`.
- mount: `getTrends()` + 초기 `getMonthSnapshot(이번달)` + `registrations`(1회 로드) 로드.
- 월 변경: `getMonthSnapshot(선택달)`만 재조회(`snapLoading`).
- 새로고침: 전체 재조회.
- 렌더: 월 선택기 → 월별 현황(헤더+KPI+도넛 4개) → 6개월 추세(헤더+막대 3개).
- 이탈(현황 KPI) = `trends.churnByMonth[key] ?? 0`.
- 디자인 토큰·팔레트·`won()` 포매터 기존 그대로. 환불 KPI/차트는 중립색(`#A7A7AA`)로(상태색 토큰 오용 금지).

## 10. 테스트

- `parseAggregateBlock`: 만원→원 환산, 음수 환불 절대값, 0 방식 제외, 소수 처리, 라벨 매핑 단위 테스트.
- 기존 순수 함수 테스트(매출 증감·이탈·직업 등) 유지.
- `getTrends`/`getMonthSnapshot`은 IO라 단위 테스트 제외(컴포넌트 수동 확인).

## 10.5 이탈(churn) 로직 전면 교체 — 디버깅 결과 반영

기존 v1의 '시트 D열(요일·시간) 빈칸 = 종료' 방식은 **근본적으로 틀려서 폐기**했다.
- 종료 버튼(`clearStudentScheduleAllSheets`)이 **모든 달 시트의 D열을 지운다** → 종료 날짜가 시트에 안 남고, 4월에 종료해도 1월 D가 비어버린다.
- 옛 알고리즘은 'D 빈 사람'을 **가장 오래된 달(1월)에 몰아서** 집계 → 라이브 확인 시 1월에 12명이 잘못 잡힘.
- D 빈칸은 종료 외 사유(등록만/스케줄 미기재)로도 생겨 오탐.

**새 정의 (`computeChurnByMonth`, 순수 함수, 단위 테스트):**
- 이탈 = **오늘 이후까지 가는 등록이 없는 수강생**. 이탈 월 = 그 사람의 **마지막 종료날짜(H)의 월**.
- **Firebase 종료기록(`studentTerminations`)이 있으면 그 날짜를 우선** — 코치가 중도 종료(H가 미래여도)한 경우를 정확히 그 달로.
- 표시 6개월 범위 안의 이탈만 카운트.
- 입력: 전 시트 원본 행(`getAllRawRows`, 중복 제거 없이) + 종료기록 + windowKeys + 오늘(자정 ms).
- 라이브 검증: 1월 6 / 2월 4 / 3월 2 / 4월 4 / 5월 6 / 6월 1 (종료날짜 기준 분산, 합 23명).

`computeSheetChurnByMonth`·`mergeChurn`은 제거. `getTrends`가 `computeChurnByMonth`를 사용.

## 11. 구현 중 확정/주의

1. 월 선택 범위는 최근 6개월. 더 과거가 필요하면 후속 확장(YAGNI).
2. 집계 블록 라벨은 행2에서 동적 매핑(하드코딩 회피).
3. 결제방식 금액은 집계 블록 기준(`tallyPaymentMethods`의 건수 집계와 별개). 기존 `tallyPaymentMethods`는 더 이상 차트에 쓰지 않으면 제거 검토(다른 사용처 없으면 삭제).
