# 매출·통계 대시보드 설계 (Revenue & Analytics Dashboard)

작성일: 2026-06-11
대상 앱: 근력학교 수강 관리 시스템 (timetable-manager)

## 1. 목적

관장(코치)이 사업 운영에 필요한 핵심 지표를 한 화면에서 확인할 수 있는 **코치 전용 통계 페이지**를 추가한다. 매출과 매출 증감을 중심으로, 수강생 구성·유입·이탈을 시각화한다.

## 4. 데이터 계층

### 4.0 신규 서비스 `src/services/analyticsService.js`
대시보드 집계 로직을 한 곳에 모은다. 시트/Firebase 원본 접근은 기존 `googleSheetsService.js` / `firebaseService.js` 함수를 재사용한다.

핵심 함수(안):
- `getMonthlyRevenue(year, month)` — 합계 셀 단일 읽기
- `getRevenueTrend(months = 6)` — 최근 N개월 매출 배열 + 전월 대비 증감
- `getStudentComposition(year, month)` — 총원/신규/재등록/남녀/직업/결제방식 집계
- `getReferralBreakdown(months = 6)` — Firebase 유입경로 집계
- `getChurn(months = 6)` — 이탈(아래 4.6)
- `buildDashboard(months = 6)` — 위 항목을 한 번에 모아 화면용 객체로 반환

### 4.1 매출 (합계 셀 읽기)
- 컬럼 I 직접 합산하지 않고, 시트에 이미 계산된 **월 매출 합계 셀**을 단일 셀로 읽는다.
- 패턴: `readSheetData(\`${getSheetNameByYearMonth(y, m)}!F1\`)` → `rows?.[0]?.[0]`
- F1 / AF3 중 실제 합계가 있는 셀은 **구현 첫 단계에서 라이브 시트를 1회 읽어 확정**하고 이 문서에 반영한다. (둘 다 비어 있으면 컬럼 I 합산으로 폴백)

### 4.2 매출 증감
- 최근 6개월(이번 달 포함) 합계 배열을 만들고, 각 달에 대해 `(당월 − 전월)` 및 `증감률(%)`을 계산.
- 가장 이른 달은 전월 데이터가 없으면 증감 표시 생략.

### 4.3 결제방식 (매출 발생처)
- 당월 시트 행을 `getAllStudents(y, m)` 로 읽어 L열 값으로 그룹핑, 각 그룹의 결제금액(I) 합과 건수 산출.

### 4.4 총원/신규/재등록/남녀
- 총원: G·H열 기간으로 오늘 기준 활성 인원(`studentRegistrationCoversDate` 로직 재사용), 같은 이름 중복은 활성 등록 1건으로 dedupe.
- 신규/재등록: 월별 F열 카운트.
- 남녀: Q열 정규화(`남`/`여`) 후 비율. (Ranking.jsx 의 기존 정규화 패턴 참고)

### 4.5 직업 (키워드 자동 그룹핑, 앱 내 처리)
- 구글 시트는 **변경하지 않는다.** R열 자유 텍스트를 읽어 앱에서 카테고리로 분류.
- 키워드 매핑 테이블(예시, 구현 시 조정):
  - 회사원: "회사", "직장", "사무", "직장인"
  - 자영업: "사업", "자영", "대표", "사장"
  - 전문직: "의사", "변호사", "약사", "교사", "강사", "디자이너", "개발"
  - 학생: "학생", "대학", "대학생"
  - 주부: "주부", "가정"
  - 기타: 위에 안 잡히는 모든 값 (빈 값 포함)
- 매핑 테이블은 `analyticsService.js` 내 상수로 두고 한 곳에서 관리.

### 4.6 이탈 (시트 + Firebase 병행)

이탈 = **코치가 종료 버튼을 누른 수강생**만 카운트. (자연 만료는 제외)

배경: 종료 버튼(`handleEndClass` → `clearStudentScheduleAllSheets`)은 모든 시트에서 해당 수강생의 **D열(요일·시간)만 비운다.** D열을 비우는 동작은 종료 버튼뿐이다. Firebase·시트 어디에도 종료 날짜 기록은 없다.

#### (a) 과거 ~ 직전 달: 시트 기반 (날짜 추정 가능)
각 월 **M**에 대해, 다음 달(M+1) 시트가 존재하면:
- **M월 이탈** = M월 시트에 행이 있고 **D열(요일·시간)이 비어 있으며**, M+1월 이후에 **활성 등록(D열이 채워진 행)이 없는** 수강생.
- D열이 비어 있어야 한다는 조건 → 만료가 아닌 **종료 버튼 처리됨**을 보장.
- 마지막으로 행이 남은 달 = 이탈 시점 → 월별 그래프에 반영.
- 이름 기준 dedupe (한 사람이 여러 달에 걸쳐 잡히지 않도록, "마지막 등록 달"에만 1회 집계).

#### (b) 이번 달(가장 최근, 다음 달 시트 없음): Firebase 기반
- `handleEndClass` 에 **Firebase 종료 기록 추가**: 컬렉션 `studentTerminations`(신규) 에 `{ studentName, terminatedAt, terminatedBy: 'coach', reason? }` 문서 생성. 기존 D열 비우기 동작은 그대로 유지.
- 이번 달(다음 달 시트가 아직 없는 가장 최근 달) 이탈은 이 Firebase 기록으로 즉시 집계.

#### (c) 중복 방지 경계
- 시트 방식은 **다음 달 시트가 존재하는 달까지만** 적용.
- Firebase 방식은 **가장 최근(다음 달 시트가 없는) 달**만 적용.
- 두 방식의 적용 월이 겹치지 않으므로 한 이탈이 양쪽에서 중복 집계되지 않는다.

### 4.7 신규 유입 경로
- `NewStudentRegistration.jsx`(7단계 위자드) 개인정보 단계에 1문항 추가: "어떻게 알고 오셨나요?"
  - 선택지: 인스타그램 / 네이버 / 지인추천 / 직접방문 / 기타
- `createNewStudentRegistration()` 데이터 객체에 `referralSource` 추가 → Firestore `newStudentRegistrations` 에만 저장. **구글 시트 컬럼(A~R) 변경 없음.**
- 대시보드는 `newStudentRegistrations` 를 기간으로 조회해 `referralSource` 그룹핑. 값이 없는 과거 신청은 "미입력"으로 분류.

## 6. 성능

- 진입 시 최근 6개월 시트 6회 + 합계 셀 6회 + Firebase 2~3 쿼리 → 1회 집계 후 화면 표시.
- 매 렌더 자동 재계산하지 않고, 결과를 컴포넌트 state 에 보관 + **새로고침 버튼**으로만 재요청.
- 로딩 중 스켈레톤/스피너 표시.
