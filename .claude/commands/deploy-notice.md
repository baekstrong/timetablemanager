---
description: 배포 후 관리자봇 업데이트 공지 게시 (초안 → 백관장 승인 → 게시)
---

방금 배포(main 푸시)된 변경을 기준으로 게시판에 관리자봇 업데이트 공지를 올리는 워크플로.

## 절차 (순서 엄수)

1. **변경 파악**: `git log --oneline -20`과 최근 머지 커밋 내용으로 이번 배포에 포함된 변경을 파악한다. 그중 **수강생이 체감하는 변경**(새 기능, 화면/동작 변화)만 추린다.
   - 체감 변경이 없으면(내부 리팩토링·마이너 버그 픽스뿐이면) "공지할 변경이 없습니다"라고 보고하고 종료한다.
2. **초안 제시**: 공지 초안(제목 + 본문)을 작성해 사용자에게 보여주고 승인을 기다린다.
   - 본문은 수강생 눈높이로: 기능별 번호 매기기, 사용 방법 위주, 존댓말.
   - **승인 전에는 절대 게시하지 않는다.** 초안은 반드시 턴의 마지막 일반 텍스트로 출력해 사용자에게 보이게 한다 (툴 호출 사이 텍스트는 안 보일 수 있음).
3. **게시**: 승인받으면 실행한다. $ARGUMENTS가 있으면 초안 작성 시 참고한다.

```bash
node --env-file=.env scripts/post-update-notice.js "제목" "본문" --unpin-old
```

- **`--unpin-old`가 기본(백관장 선호)**: 기존 관리자봇 공지는 **삭제하지 않고 상단 고정만 해제**한다. 지난 공지는 게시판/공지 필터에 기록으로 남고, 상단 고정 공지는 이번 새 공지 1건만 남는다.
  - 플래그 없이 실행하면 기존 공지를 소프트 삭제(게시판에서 사라짐)하므로, 특별히 지우려는 게 아니면 항상 `--unpin-old`를 붙인다.
- 새 공지에는 isCoach(이름 하늘색)·pinned(노란 배경+상단 고정) 필드가 자동 포함된다.
- 게시판 표시 로직: `pinned && category==='notice'`인 글만 상단 고정 배너로 뽑힌다(`firebaseService.js`). 그래서 고정만 해제하면 글은 남고 배너에서만 빠진다.

4. **결과 보고**: 고정 해제된 지난 공지 건수와 게시된 새 글의 제목·ID를 보고한다. 거절된 경우 공지 없이 종료한다.
